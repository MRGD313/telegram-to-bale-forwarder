# From-zero E2E test plan

## Scope

| Profile | Telegram source | Bale destination | Fresh DB |
|---------|-----------------|------------------|----------|
| **public** | `@Mofidot` | `@mofidottest` | `state_test_public.db` |
| **private** | invite group, **topic 4 only** («هایلایت») | `@highlights_iomi` | `state_test_private.db` |

## Preconditions

- `session` authorized (Telegram login done once)
- Bale bot is admin on target Bale channels
- `ffmpeg` on PATH
- Network access to Telegram + Bale

## Test env

Generated by `py scripts/generate_test_env.py`:

- `.env.test.public` — 8 oldest messages (stops before ~12 MB MP3 at msg 12 that times out on Bale)
- `.env.test.private` — 12 messages in topic 4 only (crawl filters by topic), strict routing, no `FORCE_BALE_CHAT`

## Steps (automated: `scripts/run_e2e_from_zero.ps1`)

### Public

1. Delete `state_test_public.db` (clean slate)
2. `MODE=crawl_then_send` with `BACKFILL_LIMIT=8`, `RESET_SENT_ON_SEND_START=1`
3. **Verify:** `py scripts/verify_test_queue.py public`
   - All rows `sent` or `skipped` (no `pending`/`failed`)
   - Sent `tg_message_id` strictly increasing (chronological)
4. **Manual:** spot-check `@mofidottest` on Bale (albums, voice, text)

### Private (topic 4)

1. Delete `state_test_private.db`
2. Crawl+send with `INCLUDE_SEND_TOPIC_IDS=4` (12 messages in that topic)
3. **Verify:** `py scripts/verify_test_queue.py private`
   - No active rows outside `topic_id=4`
   - Sends go to `@highlights_iomi` (not fallback)
4. **Manual:** spot-check `@highlights_iomi` on Bale

## Pass criteria

- Queue verifier prints `RESULT: PASS` for both profiles
- Logs contain no `STRICT_STOP permanent`
- No `FORCE_BALE_CHAT` redirect lines in test logs

## Fail actions

- If `failed` on large audio: confirm `AUDIO_REENCODE_MIN_MB=12` and re-run send only (`MODE=send`, `RESET=0`)
- If topic unmapped: run `MODE=discover_topics` and fix `TOPIC_TO_BALE_MAPPING`
